## Reporting Security Issues

Please see our [Security Policy](../../SECURITY.md) for information on how to report security vulnerabilities.

## Security Update Process

1. **Disclosure**: Security researcher reports vulnerability privately
2. **Verification**: Development team confirms and assesses severity
3. **Fix Development**: Patch is developed and tested
4. **Advisory Creation**: Detailed advisory is written
5. **Release**: Fix is released with security advisory
6. **Notification**: Users are notified via changelog and GitHub

## Stay Informed

- Watch this repository for security updates
- Check the [CHANGELOG](../../CHANGELOG.md) regularly
- Subscribe to GitHub Security Advisories for this repository
